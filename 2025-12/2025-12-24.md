### GIT에 대한 설명
<img width="893" height="288" alt="image" src="https://github.com/user-attachments/assets/3d6ae455-2abf-42e9-906d-8323c624563a" />

- GIT 이란? 분산 저장 관리 툴로서 중압 집중형 형상 관리 툴로 유명했던 SVN을 밀어내고 현재 전 세계에서 가장 유명하게 사용되는 형상관리 툴이다.
  - SVN이란? 중앙 집중형 형상 관리 툴로서 한 곳에 데이터가 저장된다. 만약 SVN을 저장하고 있는 데이터 센터 혹은 서버가 죽게 되면 데이터가 유실될 수 있다.
  - GIT의 경우에는 각 저장소가 분산이 되어 저장이 되므로 데이터 유실에 안전하다.
```
git config --global user.name = ""
git config --global user.email = ""
```
- git은 github의 저장되어 있는 이름과 이메일이 동일해야 한다.
  <img width="1109" height="144" alt="image" src="https://github.com/user-attachments/assets/f2db1ced-1067-4135-bfc4-5cd3f4237d04" />
- GIT에는 3가지 영역으로 나뉘게 되는데
  - Working Direcotry, Staging Area, Local Repository로 나뉘게 된다.
- 영역	설명	비유
  - Working Directory	실제로 파일을 편집하는 공간	책상 위
  - Staging Area	커밋할 파일을 준비하는 공간	택배 박스
  - Repository	변경 이력이 저장되는 공간	창고
- 커밋하고 싶은 파일의 경우 다음과 같은 명령어를 입력 해 사용한다.
```
git add 파일 명
```
- 실제 커밋을 할 경우에는 다음과 같은 명령어를 사용한다.
```
git commit -m ""
```

## Branch란?
- Branch는 포인터라고 볼 수 있다. 즉 특정 커밋을 가리키는 표지판이라고 보면 된다.
- 새 branch를 생성할 경우 같은 커밋을 가리키는 포인터를 하나추가하므로 가볍다.
- 커밋하면 현재 브렌치 포인터만 새로운 커밋으로 이동한다.
```
  git branch feature/temp
```
- branch로 만든 새 브렌치로 가기 위해서는 switch 명령어를 사용해야 한다.
```
  git switch feature/temp
```
- 위 명령어가 귀찮다면 git switch -c 브렌치명으로 만들수 있다.

# 원격 저장소 (Remote Repository)
<img width="399" height="509" alt="image" src="https://github.com/user-attachments/assets/fb7278a0-b591-4c26-8b6f-777fb48cd431" />


- 원격 저장소는 인터넷이나 네트워크에 있는 git 저장소
```
 git remote add origin [https://github.com/username/my-project.git](https://github.com/username/my-project.git) 
```
- 이름이 origin인 이유는 기본 원격 저장소 이름
- push의 경우에는 다음과 같은 명령어 사용 (pusy란 원격 저장소에 현재 로컬 저장소에 저장된 내용을 반영하는 것 이다.)
```
  git push -u origin main
```
- 여기서 -u는 -set--upstream을 말하며 로컬 저장소와 원격 저장소를 연결한다는 의미다.
- 만약 작업ㅇ르 하다가 저장소에 최신의 파일이 커밋되게 되면 push를 거부할 수 있으므로 git pull 명령어를 통해 불러와야 한다.
